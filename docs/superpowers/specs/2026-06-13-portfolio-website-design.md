# Сайт-визитка Евгения Заболотнего: дизайн-спецификация

## Цель

Создать одностраничный двуязычный сайт-визитку Senior Mobile Developer на React и
TypeScript. Сайт должен делать акцент на практических навыках, production-опыте и
выполненных проектах, хорошо работать на мобильных устройствах и быть готовым к
развёртыванию на статическом хостинге.

## Аудитория

- рекрутеры и технические руководители;
- заказчики проектной мобильной разработки;
- российские и международные компании.

## Технологии

- React и TypeScript;
- Vite для разработки и production-сборки;
- Three.js через React Three Fiber;
- React Three Drei для вспомогательных 3D-компонентов и адаптации производительности;
- CSS Modules или обычный CSS с дизайн-токенами, без UI-фреймворка;
- ESLint для статической проверки.

Backend, CMS и форма отправки сообщений в первую версию не входят.

## Визуальное направление

Выбрано направление **Digital Signal**:

- почти чёрный фон с фиолетовыми и холодными световыми акцентами;
- крупная современная типографика и высокая контрастность;
- тонкая координатная сетка и полупрозрачные панели;
- один узнаваемый визуальный приём: реактивное поле частиц;
- сдержанные появления секций при прокрутке;
- акцент на цифрах, технологическом стеке и результатах проектов.

Интерфейс не должен выглядеть как стандартный шаблон портфолио. Декоративные эффекты
не конкурируют с текстом и не затрудняют чтение.

## Локализация

Сайт поддерживает русский и английский языки.

- весь отображаемый контент хранится в типизированных словарях;
- язык переключается без перезагрузки;
- выбор сохраняется в `localStorage`;
- при первом посещении используется русский язык;
- `lang` у документа обновляется при переключении;
- названия продуктов и технологий не переводятся;
- обе версии имеют одинаковую структуру и набор фактов.

## Анимация и Three.js

Hero содержит поле частиц с мягким свечением и глубиной. Частицы медленно движутся,
реагируют на положение указателя и не требуют действий пользователя.

Ограничения:

- текст и CTA располагаются поверх отдельного контрастного слоя;
- сцена не блокирует прокрутку и клики;
- на мобильных уменьшаются плотность частиц, DPR и сложность шейдеров;
- производительность адаптируется по среднему FPS;
- при ошибке WebGL показывается статичный градиентный фон;
- при `prefers-reduced-motion: reduce` непрерывное движение отключается;
- тяжёлые 3D-модели и внешние текстуры в первую версию не входят.

## Адаптивность

Проектирование ведётся mobile-first.

- основные контрольные ширины: мобильная, планшетная и desktop;
- на мобильных контент строится в одну колонку;
- hero сохраняет читаемость на высоте небольших экранов;
- навигация превращается в доступное компактное меню;
- карточки проектов не требуют hover для доступа к информации;
- интерактивные элементы имеют удобную область касания;
- длинные названия технологий корректно переносятся;
- мок-изображения используют стабильное соотношение сторон и не вызывают layout shift.

## Ошибки и деградация

- отсутствие WebGL не скрывает текст hero;
- ошибка загрузки изображения показывает стилизованный placeholder;
- недоступный `localStorage` не мешает переключать язык в текущей сессии;
- ссылки email и Telegram остаются обычными ссылками;
- PDF хранится в публичных ассетах и доступен без JavaScript.

## SEO и статический деплой

Проект должен собираться в директорию `dist` и разворачиваться как статический сайт на
Vercel. Архитектура также совместима с Netlify и Cloudflare Pages.

В первую версию входят:

- уникальные `title` и `description`;
- Open Graph metadata;
- favicon и web app icons;
- canonical URL, который настраивается перед production-деплоем;
- `robots.txt`;
- `sitemap.xml`;
- понятный URL без обязательного client-side routing;
- локальные оптимизированные ассеты;
- README с командами разработки, сборки и деплоя.

Аналитика и собственный домен не входят в первую версию, но могут быть подключены после
первого развёртывания без изменения архитектуры.

## Проверка качества

Автоматические unit-тесты не требуются для первой версии из-за небольшого объёма
логики. Перед завершением обязательны:

- успешная проверка TypeScript;
- успешная проверка ESLint;
- успешная production-сборка;
- ручная проверка desktop, tablet и mobile;
- проверка переключения RU / EN и сохранения выбора;
- проверка email, Telegram и скачивания PDF;
- проверка клавиатурной навигации;
- проверка режима reduced motion;
- проверка статичного fallback без WebGL;
- проверка отсутствия критических ошибок в консоли;
- базовая проверка Lighthouse по accessibility, SEO и производительности.

## Критерии готовности

Сайт считается готовым, когда:

1. содержание соответствует резюме и не содержит выдуманных достижений;
2. четыре ключевых проекта представлены подробно, остальные — компактно;
3. RU и EN версии полностью доступны;
4. desktop и mobile версии сохраняют визуальную иерархию;
5. Three.js-сцена работает плавно и корректно деградирует;
6. PDF скачивается, email и Telegram открываются корректно;
7. production-сборка готова к публикации на Vercel;
8. мок-изображения можно заменить реальными без переработки компонентов.
